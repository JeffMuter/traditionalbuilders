# AGENTS.md

Guide for AI agents working in the Traditional Builders codebase.

## Project Overview

Traditional Builders is a web platform connecting clients with traditional architects and builders, featuring directory listings, project cost estimation, and portfolio browsing. The platform emphasizes timeless design principles and quality craftsmanship.

**Tech Stack:**
- **Backend**: Go 1.25+ with standard library HTTP server
- **Templates**: [templ](https://templ.guide/) for type-safe HTML templating
- **Database**: SQLite 3 with Goose migrations
- **Styling**: Tailwind CSS
- **Frontend Interactivity**: htmx 2.0.4 (CDN)
- **Dev Environment**: Nix shell (direnv-enabled)

## Code Organization & Patterns

### Entry Point (cmd/server/main.go)

- Uses `database/sql` with SQLite driver (`github.com/mattn/go-sqlite3`)
- Database connection initialized first, with ping check
- Routes registered on `http.NewServeMux`
- Static file serving via `http.FileServer` at `/static/` prefix
- Server runs on port 8080
- Fatal errors use `log.Fatal`

### Handlers (internal/handlers/)

**Pattern:**
```go
func HandlerName(w http.ResponseWriter, r *http.Request) {
    templates.TemplateName().Render(r.Context(), w)
}
```

- Handlers are simple functions matching `http.HandlerFunc` signature
- Call templ-generated template functions directly
- Pass `r.Context()` to template rendering for proper context handling
- Import path: `github.com/emerald/traditionbuilders/templates`

### Templates (templates/)

**File Naming:**
- Source files: `name.templ`
- Generated files: `name_templ.go` (gitignored, auto-generated by `templ generate`)

**Pattern:**
```templ
package templates

templ ComponentName() {
    <div class="tailwind-classes">
        // HTML content
    </div>
}

templ PageName() {
    <!DOCTYPE html>
    <html lang="en">
        <head>
            <meta charset="UTF-8"/>
            <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
            <title>Page Title</title>
            <link rel="stylesheet" href="/static/css/output.css"/>
        </head>
        <body>
            // Page content
        </body>
    </html>
}
```

**Templ Guidelines:**
- All templates in `package templates`
- Use `templ ComponentName()` syntax for component definitions
- Reference static CSS: `/static/css/output.css`
- Full HTML pages include complete `<!DOCTYPE html>` structure
- Self-closing tags use XML syntax: `<meta charset="UTF-8"/>`

### Database

**Connection:**
- SQLite database file: `traditionbuilders.db` (created by migrations)
- Driver: `github.com/mattn/go-sqlite3` (imported with blank identifier `_`)
- Connection string: `./traditionbuilders.db`

**Migrations:**
- Location: `db/migrations/`
- Tool: Goose
- Naming: `NNN_description.sql` (e.g., `001_init.sql`)
- Format: SQL files with `-- +goose Up` and `-- +goose Down` markers

**Schema (current):**
```sql
professionals (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    phone TEXT,
    specialty TEXT NOT NULL,
    location TEXT,
    bio TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
)

projects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    professional_id INTEGER (FK -> professionals.id),
    title TEXT NOT NULL,
    description TEXT,
    location TEXT,
    cost_estimate INTEGER,
    completed_at DATE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

### Styling

**Framework:** Tailwind CSS

**Color Palette:**
- Primary: `stone-*` shades (stone-50, stone-600, stone-800, stone-900)
- Background: `bg-stone-50` (off-white), `bg-white`
- Text: `text-stone-*`
- Accents: `bg-stone-800` (dark buttons/headers)

**Typography:**
- Headings: `font-serif` (classical feel)
- Body: default sans-serif
- Large headings: `text-5xl font-serif font-bold`
- Subheadings: `text-xl font-serif font-bold`

**Layout Patterns:**
- Container: `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8`
- Responsive grids: `grid md:grid-cols-3 gap-8`
- Flexbox for alignment: `flex justify-between items-center`

**Configuration:**
- `tailwind.config.js` scans `./templates/**/*.templ`
- Input: `static/css/input.css` (contains Tailwind directives)
- Output: `static/css/output.css` (generated, gitignored)

### Frontend Interactivity

**Framework:** htmx 2.0.4

**Integration:**
- Loaded via CDN in all page templates: `<script src="https://unpkg.com/htmx.org@2.0.4" ...></script>`
- Enables declarative AJAX, CSS transitions, WebSockets, and server-sent events via HTML attributes
- No build step required

**Common htmx Patterns:**
- `hx-get="/api/endpoint"` - Issue GET request
- `hx-post="/api/endpoint"` - Issue POST request
- `hx-trigger="click, keyup delay:300ms"` - Event triggers
- `hx-target="#result"` - Where to insert response
- `hx-swap="innerHTML"` - How to swap content (innerHTML, outerHTML, beforeend, etc.)
- `hx-indicator="#spinner"` - Show loading state

**Usage Guidelines:**
- Use htmx for dynamic updates without full page reloads
- Handlers can return HTML fragments (templ components) for htmx requests
- Check `r.Header.Get("HX-Request")` to detect htmx requests vs full page loads
- Return appropriate content: full page for normal requests, fragments for htmx
- Leverage Go's templ components to render partial HTML responses

**Resources:**
- [htmx Documentation](https://htmx.org/docs/)
- [htmx Examples](https://htmx.org/examples/)

## Common Gotchas

### Templ Template Changes Not Reflected
**Symptom:** Changed `.templ` file but no changes in browser  
**Solution:** Run `templ generate` before restarting server (or use `make dev`/`run`)

### CSS Changes Not Reflected
**Symptom:** Changed Tailwind classes but styles don't update  
**Solution:** Rebuild CSS with `make css` or run `make css-watch` in separate terminal

### Database Errors on Startup
**Symptom:** `database is locked` or migration errors  
**Solution:** Run `make migrate` to ensure migrations are current. If corrupt, `make clean` and `make setup`

### Import Errors for Templates
**Symptom:** Cannot import `templates` package  
**Solution:** Templates must be generated first. Run `templ generate`

### Nix Environment Not Active
**Symptom:** Commands like `goose`, `templ` not found  
**Solution:** Activate nix-shell with `nix-shell` or ensure direnv is enabled

### Port Already in Use
**Symptom:** Server fails to start with address already in use  
**Solution:** Server runs on port 8080. Kill existing process or change port in `cmd/server/main.go`

## Adding New Features

### New Page/Route
1. Create `.templ` file in `templates/` with full HTML structure
2. Run `templ generate` to create Go code
3. Add handler function in `internal/handlers/` that calls template
4. Register route in `cmd/server/main.go` with `mux.HandleFunc`
5. Test the endpoint

### New Database Table
1. Create new migration file: `db/migrations/NNN_description.sql`
2. Add `-- +goose Up` section with CREATE TABLE
3. Add `-- +goose Down` section with DROP TABLE
4. Run `make migrate`

### New Component
1. Add `templ ComponentName()` in existing or new `.templ` file
2. Run `templ generate`
3. Use component in other templates: `@ComponentName()`

### Adding Dependencies
1. Import package in Go code
2. Run `go mod tidy` to update `go.mod` and `go.sum`
3. If tool is required (not Go package), add to `buildInputs` in `shell.nix`

## Code Style

### Go
- Standard Go formatting (gofmt)
- Explicit error handling (check every error)
- Use standard library where possible
- Simple, straightforward patterns
- HTTP handlers follow `func(w http.ResponseWriter, r *http.Request)` signature

### Templ
- PascalCase for template names: `Landing()`, `ProfileCard()`
- Use Go-style formatting for embedded expressions
- Keep templates focused (split into components as needed)
- Full pages have complete HTML structure; components are fragments

### SQL
- `snake_case` for table and column names
- `id INTEGER PRIMARY KEY AUTOINCREMENT` for primary keys
- Timestamps: `created_at`, `updated_at` with `DATETIME DEFAULT CURRENT_TIMESTAMP`
- Foreign keys explicitly defined with `FOREIGN KEY ... REFERENCES`

### Naming Conventions
- Handlers: Verb or noun matching the page/action (e.g., `Landing`, `ListProfessionals`)
- Templates: Match handler names
- Files: `lowercase.ext` or `lowercase_with_underscores.ext`
- Go packages: lowercase, single word if possible

## Workflow Best Practices

### Before Making Changes
1. Ensure nix-shell is active
2. Verify database is initialized: `make migrate-status`
3. Run `make setup` if starting fresh

### During Development
1. Use `make dev` or `run` for automatic rebuilds
2. Or run `make css-watch` in background and `go run cmd/server/main.go` manually
3. Edit `.templ` files, run `templ generate`, refresh browser
4. Test changes with `make test`

### Before Committing
1. Ensure generated files are not staged (they're gitignored)
2. Run `make test` to verify tests pass
3. Verify `make build` succeeds
4. Check that database migrations are included if schema changed

### When Changing Templates
1. Edit `.templ` file
2. Run `templ generate` (or let `make dev`/`run` do it)
3. Verify `*_templ.go` is regenerated (timestamp check)
4. Restart server to load new code

### When Changing CSS
1. Edit Tailwind classes in `.templ` files
2. Run `make css` or have `make css-watch` running
3. Hard refresh browser (Ctrl+Shift+R) to bypass cache
